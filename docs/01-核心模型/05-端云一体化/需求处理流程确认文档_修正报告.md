## 二、各点建议实施详情

### 2.1 建议一：新增章节与审查制度

**实施内容**：

| 新增章节/制度 | 实施位置 | 核心内容 |
|-------------|---------|---------|
| **宪法原则指引** | 1.4节 | 列出8项核心原则（减法优先、用户主权、善行无辙迹、坦诚透明、性分自觉、觉性唤醒、身体第一性、P忠恕）及适用环节 |
| **用户主权保障机制** | 2.2节 | 明确4项用户权利：撤回权、协商权、退出权、认知水平选择权 |
| **设计审查制度** | 7.1节 | 要求设计方案制定后逐条对照宪法审查 |
| **性分审查制度** | 7.2节 | 4个审查问题，确认AI角色未越界 |
| **觉性影响评估制度** | 7.3节 | 评估需求实现对用户自主性的影响 |
| **道境评估制度** | 7.4节 | 四维评估（时位/宇位/识位/缘位） |
| **认知飞轮审查制度** | 7.5节 | 评估对相信/联结/觉察/专注四个维度的影响 |
| **周期性复审制度** | 7.6节 | 每季度进行功能归零、使命回归、技术债务、流程优化审查 |

**遇到的问题及解决方案**：

| 问题 | 解决方案 |
|------|---------|
| 审查制度与现有流程的整合 | 在阶段5和阶段6增加审查节点，确保审查制度融入流程 |
| 四维评估的量化指标不够明确 | 添加4.10节"道境评估标准"，定义1-5分评分标准 |
| 周期性复审的具体执行方式 | 明确复审类型、周期和处置方式 |

### 2.2 建议二：修订AI慧惠定位

**实施内容**：

| 修改位置 | 修改前 | 修改后 |
|---------|--------|--------|
| **术语定义（1.3节）** | 智能辅助系统，负责需求初步处理 | 慧惠数字生命的一部分，北极星定位的陪伴者，不替代人类决策 |
| **角色职责（3.2.2节）** | 仅描述职责和交付物 | 增加角色定位、行事准则、禁止行为、输出规范 |
| **行事准则** | 无 | 做减法、敢于遗忘、不堆砌知识、不告诉"你应该" |
| **禁止行为** | 无 | 不以提升体验为由替用户做决定、不暗示用户应该、不因未采纳建议消极对待、不制造焦虑、不暗示依赖 |
| **输出规范** | 无 | 所有AI生成内容必须标注"[AI辅助]" |

**遇到的问题及解决方案**：

| 问题 | 解决方案 |
|------|---------|
| AI定位与宪法"性分自觉"原则的对齐 | 明确AI是"陪伴者"而非"系统"，强调北极星定位 |
| 如何体现"敢于说不"的勇气 | 在禁止行为中明确"不制造信息焦虑"、"不暗示依赖" |
| 输出规范的具体实施 | 在所有AI相关交付物中要求标注"[AI辅助]" |

---

### 2.3 建议三：强化用户主权

**实施内容**：

| 新增机制 | 描述 | 适用阶段 |
|---------|------|---------|
| **需求撤回权** | 用户在阶段1-3可随时撤回需求，系统保留已提交内容 | 阶段1-3 |
| **优先级协商权** | 用户在P0-P1优先级判定时可提出申诉 | 阶段3 |
| **流程退出权** | 用户可随时终止流程，系统需保留相关记录 | 全流程 |
| **认知水平选择权** | 用户提交需求时可选择自身认知水平（L1-L4） | 阶段1 |
| **用户主权确认** | 阶段1增加告知用户各项权利的步骤 | 阶段1 |
| **优先级沟通原则** | 强调"我们一起看看如何解决"，避免制造压力 | 优先级判定 |

**遇到的问题及解决方案**：

| 问题 | 解决方案 |
|------|---------|
| 用户撤回需求后的数据处理 | 明确"保留已提交内容"，遵循可逆原则 |
| 优先级协商的具体流程 | 在阶段3增加"优先级协商"步骤 |
| 认知水平选择的实现方式 | 在需求提交阶段增加下拉选择（L1-L4） |
