# 旋量-太极模型知识库管理规则

> 本规则旨在建立一套全面、系统的知识库管理机制，确保项目的规范化管理、高效维护和便捷使用。  
> 生效日期：2026-06-20  
> 最后更新：2026-06-20

---

## 第一章 总则

### 1.1 规则目的

建立标准化、可追溯、可协作的知识库管理体系，确保旋量-太极模型理论体系的一致性和完整性。

### 1.2 适用范围

本规则适用于 `spinor-taiji-model` 项目下的所有文档、图片、脚本和数据文件。

### 1.3 核心原则

| 原则 | 说明 |
|------|------|
| 一致性 | 所有命名和分类遵循统一规范 |
| 可追溯 | 所有变更记录在案 |
| 可协作 | 支持多人协作和版本管理 |
| 可检索 | 建立有效的索引机制 |

## 第二章 文件分类体系

```

| 分类编号 | 分类名称 | 包含内容 | 文件类型 |
|----------|----------|----------|----------|
| 00 | 产品宪法 | 项目规范、伦理准则、宪法文档 | .md, .pdf |

| 05 | 五步读解法 | 方法论、流程图、案例 | .md |
| 09 | 参考文献 | 学术文献、引用资料 | .pdf, .md |
| 10 | 词汇表 | 术语定义、维护规范、变更日志 | .md |

## 第三章 文件命名规范

### 3.1 命名规则总表

| 文档类型 | 命名规则 | 示例 | 说明 |
|----------|----------|------|------|
| 模型理论 | 中文核心概念 | `旋量与太极的关系.md` | 直接使用理论名称 |
| 经典读解 | `XXX-章节名.md` | `055-含德之厚与赤子.md` | 章节号+核心概念 |
| 对读分析 | `对读_章节A与章节B_主题.md` | `对读_050与055_无死地.md` | 对读专题 |
| 应用案例 | `领域_主题.md` | `长期胃病_振动病理切片.md` | 领域+具体主题 |
| 阅读地图 | `系列名_阅读地图.md` | `道德经操作手册_阅读地图.md` | 系列+阅读地图 |
| 学习报告 | `学习报告_主题.md` | `学习报告_呼吸的双重阴阳螺旋.md` | 学习过程记录 |
| 审查报告 | `审查_主题.md` | `审查功夫_知行合一学习报告.md` | 审查意见记录 |
| 质量报告 | `质量检查_文件类型.md` | `旋量与太极_质量检查报告.md` | 质量检查结果 |
| 版本说明 | `XXX_转换说明.md` | `看山三境界_转换说明.md` | 版本转换记录 |
| 提示词 | `XXX_提示词.md` | `万物振动_头图提示词.md` | 生成提示词 |
| 配图方案 | `配图方案_文章名.md` | `配图方案_长期胃病公众号.md` | 配图规划 |

### 3.2 版本后缀规范

| 后缀 | 含义 | 使用场景 |
|------|------|----------|
| `_v1.0.0` | 正式版本 | 初始发布 |
| `_v2.0.0` | 重大更新 | 架构调整 |
| `_公众号版` | 公众号专用 | 发布版本 |
| `_公众号专业版` | 专业版格式 | 带完整排版 |
| `_校准版` | 术语校准 | 定义统一后 |
| `_修复版` | Bug修复 | 问题修复后 |
| `_优化版` | 性能优化 | 优化后版本 |
| `_学术排版版` | 学术格式 | 学术论文用 |
| `_案例互动版` | 互动格式 | 含互动元素 |

### 3.3 命名禁则

| 禁则 | 说明 | 正确做法 |
|------|------|----------|
| 禁用空格 | 路径解析问题 | 使用下划线 `_` |
| 禁用特殊字符 | 文件系统兼容 | 使用中文或英文 |
| 禁用过长名称 | 可读性差 | 控制在50字以内 |
| 禁用无意义编号 | 难于理解 | 使用描述性名称 |
| 避免重复命名 | 版本混乱 | 添加版本后缀 |

## 第五章 版本控制机制

### 5.1 版本号规范

```
主版本号.次版本号.修订号

- 主版本号：重大架构调整或理论突破
- 次版本号：新增重要内容或功能
- 修订号：小幅修改或错误修正
```

### 5.2 版本标记规则

| 版本类型 | 触发条件 | 示例 |
|----------|----------|------|
| v1.0.0 | 初始版本发布 | 首次完整定义 |
| v1.1.0 | 新增章节解读 | 新增第58章解读 |
| v1.1.1 | 修正定义错误 | 术语校准 |
| v2.0.0 | 理论重大更新 | 核心模型重构 |

### 5.3 Git提交规范

```
<类型>: <简短描述>

类型：
- feat: 新增内容
- fix: 修复问题
- docs: 文档更新
- style: 格式调整
- refactor: 重构
- test: 测试相关

示例：
feat: 新增第58章道德经解读
fix: 修正旋量定义中的笔误
docs: 更新README目录结构
```

### 5.4 分支管理

| 分支 | 用途 | 命名规则 |
|------|------|----------|
| main | 主分支，稳定版本 | main |
| develop | 开发分支 | develop |
| feature | 功能开发 | feature/功能名 |
| hotfix | 紧急修复 | hotfix/问题名 |

## 第六章 内容更新流程

### 6.1 词汇表更新流程

```
提议  起草  审查  状态标记  录入主表  更新日志  更新关系图
```

| 步骤 | 负责人 | 输出物 |
|------|--------|--------|
| 提议 | 任何人 | 提议记录 |
| 起草 | 提议者 | 词汇定义草稿 |
| 审查 | 审查者 | 审查意见 |
| 状态标记 | 维护者 | // |
| 录入主表 | 维护者 | 更新词汇表 |
| 更新日志 | 维护者 | 变更日志 |
| 更新关系图 | 维护者 | 词汇关系图 |

### 6.2 文章更新流程

```
创作  初审  修订  终审  发布  归档
```

| 阶段 | 内容 | 负责人 |
|------|------|--------|
| 创作 | 按命名规范起草 | 作者 |
| 初审 | 内容准确性检查 | 审查者 |
| 修订 | 根据意见修改 | 作者 |
| 终审 | 格式规范检查 | 维护者 |
| 发布 | 推送到公众号 | 发布者 |
| 归档 | 保存HTML版本 | 系统 |

### 6.3 触发更新的信号

| 信号 | 来源 | 响应 |
|------|------|------|
| 新读解完成 | 创作完成 | 通读词汇表，更新相关词条 |
| 跨文章不一致 | 审查发现 | 立刻校准并更新主表 |
| 学习者反馈 | 用户提问 | 补充关联性说明 |
| 定期审核 | 季度审核 | 批量更新和调整 |

关键词：旋量、太极、道德经、振动、呼吸
标签：理论基础、经典读解、应用实践
日期：2026-06-20

---
```

### 8.2 分类索引表

| 分类 | 关键词标签 | 关联分类 |
|------|------------|----------|

## 第九章 备份与恢复方案

### 9.1 备份策略

| 备份类型 | 频率 | 保留时间 | 存储位置 |
|----------|------|----------|----------|
| 自动备份 | 每次提交时 | 永久 | Git仓库 |
| 每日备份 | 每天 | 30天 | 本地 |
| 每周备份 | 每周 | 90天 | 外部存储 |
| 版本发布 | 每次发布 | 永久 | Git标签 |

### 9.2 恢复流程

```
发现问题  确定版本  评估影响  执行恢复  验证  记录
```

### 9.3 灾难恢复

| 场景 | 恢复方法 | 预计时间 |
|------|----------|----------|
| 文件误删 | `git restore` | < 5分钟 |
| 目录损坏 | `git checkout` | < 10分钟 |
| 全部丢失 | `git clone` | < 30分钟 |
