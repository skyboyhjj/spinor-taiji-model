### 1.2 词汇表特殊处理规则

词汇表需要移除的 DOM 元素：

| 元素 | 选择器 | 移除原因 |
|------|--------|---------|
| 固定 Header | `header` | 公众号正文无页面级导航 |
| 目录导航侧边栏 | `aside.sidebar` | 用户明确要求移除 |
| 搜索框 | `#searchInput` | 不适用于公众号静态阅读 |
| 浮动按钮组 | `.nav-buttons` | 包含返回顶部/首页/反馈按钮 |
| 反馈弹窗 | `.feedback-overlay` | 公众号不支持交互弹窗 |
| 底部反馈区 | `.feedback-section` | 公众号内无法提交反馈 |
| 固定 Footer | `footer` | 公众号正文无全局 footer |

### 1.3 保留的内容结构

| 内容 | 来源 | 说明 |
|------|------|------|
| 声明正文 | statement.html | 包含 h1/h2/h3/blockquote/table 等所有章节 |
| 词汇表正文 | glossary.html | 包含引言/版本/图形/术语卡片/快速参考表 |

### 1.4 微信公众号格式规范合规性

| 规范项 | 处理方式 |
|--------|---------|
| 最大宽度 | 容器限制 `max-width: 677px`，与公众号正文宽度一致 |
| 字体兼容性 | 使用 `-apple-system, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei"` |
| 图片 | 使用绝对宽度时设置 `max-width: 100%`，自动适配微信 |
| 深色模式 | 模板内置 `[data-darkmode-*]` 适配 |
| 行高 | 正文 `line-height: 2`，标题 `line-height: 1.2-1.6` |
| 禁止元素 | 移除所有 JavaScript、表单、iframe、flash |

### 2.2 命名规则

```
{文章标题}_{版本号}_公众号版.html
```

版本号与源文件 `meta[name="version"]` 或词汇表版本信息保持一致。

### 2.3 版本管理方案

| 策略 | 说明 |
|------|------|
| 增量发布 | 每次发布前，将旧版本移入 `_archive/` 目录 |
| 版本对照 | 在 README 或文档中记录每次发布的源文件版本与公众号版本对应关系 |
| 不使用 Git | 公众号产物为一次性发布物，不纳入 Git 仓库管理 |
| 存档命名 | `_archive/{标题}_{旧版本号}_公众号版.html` |

## 三、实施流程规划

### 3.1 分阶段计划

#### 阶段一：准备阶段（约 0.5 小时）

| 任务 | 负责人 | 产出物 |
|------|--------|--------|
| 确认源文件内容完整性 | AI | 确认声明 v1.2、词汇表 v1.1.0 内容无误 |
| 审查微信公众号样式模板 | AI | 确认模板样式符合旋量-太极品牌调性 |
| 准备转换脚本 | AI | 新建 `scripts/convert_to_wechat.py`，支持声明和词汇表两种转换模式 |
| 生成转换产物 | AI | 2 个公众号版 HTML 文件 |

#### 阶段二：转换阶段（约 1 小时）

| 任务 | 负责人 | 产出物 |
|------|--------|--------|
| 提取声明正文并注入样式 | AI | `旋量-太极模型声明_v1.2_公众号版.html` |
| 提取词汇表正文（移除导航） | AI | `旋量-太极词汇表_v1.1.0_公众号版.html` |

| 补充微信公众号规范检查 | AI | 确认无违规内容（外部链接仅允许白名单域名） |

#### 阶段三：测试阶段（约 0.5 小时）

| 任务 | 负责人 | 产出物 |
|------|--------|--------|
| 本地浏览器预览 | 用户 | 截图记录 375px / 414px / 768px 三个断点下的显示效果 |
| 微信公众号草稿箱测试 | 用户 | 使用 `convert_and_push.py` 推送至草稿箱 |
| 移动端阅读体验验证 | 用户 | 确认字号、间距、图片、表格在手机上的可读性 |
| 确认无内容截断或样式错乱 | 用户 | 逐字逐段核对源文件与转换后文件的内容一致性 |

#### 阶段四：发布阶段（约 0.5 小时）

| 任务 | 负责人 | 产出物 |
|------|--------|--------|
| 推送至微信公众号草稿箱 | 用户 | 通过草稿箱审核流程 |
| 发布声明文章 | 用户 | 微信公众号后台定时或立即发布 |
| 发布词汇表文章 | 用户 | 微信公众号后台定时或立即发布 |

### 3.2 质量检查标准

| 检查项 | 标准 |
|--------|------|
| 内容一致性 | 源文件与公众号版的文字内容完全一致（段落数量 ±0） |
| 标题层级 | h1 → h2 → h3 → h4 层级关系正确 |
| 引用块 | 左侧 4px 深红色边框，背景淡红色，无多余符号 |
| 表格 | 表头有底色，斑马纹正确，奇偶行交替 |
| 浮动元素 | 声明页：移除浮动按钮；词汇表：移除侧边栏 |
| 图片 | 宽度不超过 677px，无拉伸变形 |
| 交互元素 | 全文无 JavaScript、无表单、无弹窗 |
| 微信合规 | 无外部链接（除白名单），无诱导关注，无违规关键词 |

## 四、技术规范要求

### 4.1 微信内置浏览器兼容性

| 技术点 | 规范 |
|--------|------|
| 渲染引擎 | WebKit（iOS）/ Blink（Android），标准 HTML5/CSS3 |
| Flexbox | 使用，无前缀 |
| CSS 变量 | 支持（iOS Safari 9.3+、Android 5.0+） |
| `rem` 单位 | 避免，统一使用 `px` 和 `%` |
| `position: sticky` | 词汇表侧边栏转换时移除，因公众号正文容器无固定视口 |
| `backdrop-filter` | 移除，iOS 微信不完全支持 |

### 4.2 移动端阅读体验

| 元素 | 字号 | 行高 | 说明 |
|------|------|------|------|
| 正文段落 | 15px | 2 | 公众号标准阅读字号 |
| 一级标题 | 22px | 1.2 | 居中，带装饰边框 |
| 二级标题 | 20px | 1.2 | 左侧 4px 红色边框 |
| 三级标题 | 17px | 1.6 | 左侧 3px 红色边框 |
| 引用块 | 15px | 1.8 | 左侧 4px 边框，背景淡红 |
| 表格 | 13-15px | 1.6 | 单元格自适应宽度 |

### 4.3 微信公众号内容安全规范

| 规范 | 处理方式 |
|------|---------|
| 外部链接 | 仅允许 `spinortaiji.com` 等白名单域名，其他外链一律移除或替换为纯文本 |
| 图片 | 优先使用图床 CDN，图片 URL 需为 HTTPS |
| 诱导关注/分享 | 全文禁止出现"转发朋友圈"、"关注公众号"等诱导文案 |
| 敏感词 | 声明页和词汇表均为理论文档，无敏感词风险 |
| 原创标识 | 发布时勾选"原创"（如适用），词汇表因含图表建议声明"转载" |

## 五、转换脚本设计方案

`scripts/convert_to_wechat.py` 应支持两种模式：

| 模式 | 命令 | 说明 |
|------|------|------|
| 声明模式 | `python convert_to_wechat.py statement` | 提取声明正文，移除浮动按钮/弹窗 |
| 词汇表模式 | `python convert_to_wechat.py glossary` | 提取正文，移除 header/sidebar/搜索框/浮动按钮 |

**核心逻辑**：
1. 读取源 HTML 文件
2. 使用 BeautifulSoup 解析 DOM
3. 提取 `<main>` 或 `.content` 中的正文内容
4. 注入公众号样式模板的 `<style>` 块
5. 根据模式移除指定元素
6. 替换外部图片链接为微信兼容格式
7. 输出到 `wechat/` 目录
