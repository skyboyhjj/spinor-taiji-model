## SK001：文件名规范化技能

### 1.1 操作步骤

**步骤 1：识别问题文件**
```bash
# 列出包含中文引号的文件
Get-ChildItem articles\*.html | Where-Object { $_.Name -match "[\u201c\u201d\u300a\u300b]" }
```

**步骤 2：执行规范化脚本**
```bash
python scripts/normalize_filenames.py
```

**步骤 3：验证结果**
```bash
Get-ChildItem articles\*.html | Select-Object Name
```

### 1.2 实施标准

| 标准项 | 要求 |
|--------|------|
| 引号处理 | 移除所有中文引号（""、""、《》） |
| 特殊字符 | 不允许文件名包含中文标点符号 |
| 空格处理 | 使用下划线替代空格 |
| 文件编码 | UTF-8 编码 |

### 1.3 质量要求

| 检查项 | 验收标准 |
|--------|----------|
| 文件可访问性 | 所有文件可正常打开和读取 |
| 链接完整性 | 内部链接未被破坏 |
| 命名一致性 | 同类文件命名风格统一 |

### 1.4 验证方法

```bash
# 检查是否仍有中文引号文件
Get-ChildItem articles\*.html | Where-Object { $_.Name -match "[\u201c\u201d]" } | Measure-Object
```

## SK002：构建脚本配置化技能

### 2.1 操作步骤

**步骤 1：创建配置目录**
```bash
mkdir -p config
```

**步骤 2：定义别名映射配置**
- 编辑 `config/article_aliases.json`
- 配置项：`aliases`（文件名映射）、`redirect_rules`（重定向规则）

**步骤 3：更新构建脚本**
- 修改 `scripts/build.py` 读取外部配置文件

**步骤 4：测试构建**
```bash
python scripts/build.py
```

### 2.2 实施标准

| 配置项 | 格式要求 |
|--------|----------|
| 中文文件名 | 规范化后的文件名 |
| 英文别名 | 小写字母+连字符 |
| 重定向状态码 | 301（永久重定向） |
| 配置版本 | 语义化版本号 |

### 2.3 质量要求

| 检查项 | 验收标准 |
|--------|----------|
| 配置完整性 | 所有中文文件都有对应的英文别名 |
| 规则一致性 | 重定向规则与别名映射一致 |
| 可维护性 | 配置文件易于编辑和扩展 |

### 2.4 验证方法

```bash
# 验证配置加载
python scripts/build.py | Select-String "加载配置文件"
```

## SK003：自动化构建验证技能

### 3.1 操作步骤

**步骤 1：执行构建**
```bash
python scripts/build.py
```

**步骤 2：运行验证脚本**
```bash
python scripts/validate_build.py
```

**步骤 3：查看验证报告**
- 检查成功项、警告项、错误项
- 根据错误信息修复问题

### 3.2 实施标准

| 验证项 | 标准 |
|--------|------|
| 目录检查 | dist、articles、docs 目录存在 |
| 文件检查 | 英文别名文件存在且非空 |
| 内容检查 | 文件包含有效 HTML 内容 |
| 配置检查 | _redirects 文件存在 |

### 3.3 质量要求

| 检查项 | 验收标准 |
|--------|----------|
| 零错误 | 验证脚本返回 0 错误 |
| 警告可控 | 警告数量不超过总检查项的 20% |
| 完整性 | 所有必要文件都被验证 |

### 3.4 验证方法

```bash
# 运行完整验证流程
python scripts/build.py && python scripts/validate_build.py
```

## SK004：Cloudflare 缓存策略技能

### 4.1 操作步骤

**步骤 1：定义缓存配置**
- 编辑 `config/headers_config.json`
- 配置各类型文件的缓存策略

**步骤 2：集成到构建脚本**
- 修改 `scripts/build.py` 生成 `_headers` 文件

**步骤 3：部署验证**
- 推送代码到 GitHub
- 验证 Cloudflare Pages 部署日志

### 4.2 实施标准

| 文件类型 | 缓存策略 |
|----------|----------|
| HTML 页面 | max-age=86400（1天） |
| 首页 | max-age=600（10分钟） |
| CSS/JS | max-age=31536000（1年）+ immutable |
| 图片 | max-age=31536000（1年）+ immutable |

### 4.3 安全要求

| 安全头 | 值 | 说明 |
|--------|-----|------|
| X-Content-Type-Options | nosniff | 防止 MIME 嗅探 |
| X-Frame-Options | DENY | 防止点击劫持 |
| X-XSS-Protection | 1; mode=block | 启用 XSS 保护 |
| Referrer-Policy | strict-origin-when-cross-origin | 控制 Referrer |

### 4.4 验证方法

```bash
# 验证 _headers 文件生成
Get-Content dist/_headers | Select-Object -First 10
```
